## Searching the Catalog

### Simple Search

The easiest way to find materials is through the **simple search box** on the homepage.

**What you can search for:**
- Book titles (e.g., "Sapiens", "Pride and Prejudice")
- Author names (e.g., "Harari", "Jane Austen")
- Subjects (e.g., "History", "Romance")
- ISBN numbers (e.g., "9780062316097")
- Keywords from summaries and descriptions

**How it works:**
1. Type your search term in the box
2. Press Enter or click the Search button
3. Results appear ranked by relevance
4. The system searches across titles, authors, subjects, publishers, and summaries

**Smart features:**
- **Partial matching** - Type "histo" to find "History"
- **Automatic ranking** - Most relevant results appear first
- **Spell correction** - Get "Did you mean?" suggestions for typos
- **Full-text search** - Searches inside summaries and notes

### Advanced Search

For more precise searching, click **"Advanced Search"** on the homepage.

**Search fields available:**
1. **Title** - Search specifically in book titles
2. **Author** - Find works by specific authors (use "Last name, First name" format)
3. **Subject** - Search subject headings
4. **ISBN** - Enter ISBN-10 or ISBN-13 (with or without hyphens)
5. **Publisher** - Find books from specific publishers
6. **Publication Year** - Set date ranges (from/to)
7. **Material Type** - Filter by format

**Boolean operators:**
- **AND** (default) - All terms must appear (e.g., "history AND america" finds books about American history)
- **OR** - Any term can appear (e.g., "fiction OR novel" finds either)

**How to use:**
1. Fill in any combination of fields
2. Choose AND or OR operator
3. Click Search
4. Use the Reset button to clear all fields

**Pro tips:**
- Leave fields empty to ignore them
- Combine multiple fields for very specific searches
- Use year ranges to find contemporary vs. historical materials
- Select material type to find only books, DVDs, etc.

### Spell Correction ("Did You Mean?")

The catalog includes intelligent spell checking:

**When it activates:**
- Your search returns fewer than 5 results
- The system finds similar terms in the catalog
- Suggestions appear at the top of results

**How it works:**
1. You search for "shakspere" (misspelled)
2. System finds "shakespeare" in the catalog with 85% similarity
3. A blue suggestion box appears: "Did you mean: shakespeare?"
4. Click the suggestion to search with the corrected spelling

**Features:**
- **Multi-word correction** - Fixes each word in your query
- **Confidence scoring** - Only shows suggestions with 40%+ similarity
- **Catalog-based** - Suggests terms that actually exist in your library
- **Smart matching** - Compares against titles, authors, subjects, and publishers

### Search Results Page

After searching, you'll see a comprehensive results page with powerful filtering.

**Layout:**
- **Left sidebar** - Faceted filters for refining results
- **Main area** - List of matching records with key information
- **Top bar** - Shows result count, current page, and sort options

**Each result shows:**
- **Title and subtitle** - Full bibliographic title
- **Author** - Main author or creator
- **Publication info** - Publisher, place, date
- **Material type** - Book, DVD, serial, etc.
- **ISBN/ISSN** - When available
- **Availability** - Number of copies and status
- **Summary snippet** - First lines of the description

**Faceted filters (left sidebar):**

1. **Material Types**
   - Book, E-book, Serial, DVD, Audiobook, etc.
   - Shows count for each type in your results
   - Click to filter to only that type
   - Multiple selections allowed

2. **Languages**
   - Filter by language of publication
   - Automatically detected from catalog records

3. **Publication Years**
   - Decade-based filtering
   - Click ranges like "2020-2024" or "2010-2019"

4. **Availability**
   - Available - Can be checked out now
   - Checked Out - Currently borrowed
   - On Order - Being acquired
   - Missing/Damaged - Not available

5. **Locations**
   - Main Library, Reference, Special Collections, etc.
   - Filter by physical or virtual location

**Active filters:**
- Appear as blue tags below the search box
- Click the X on any tag to remove that filter
- "Clear All Filters" button removes everything

**Sorting options:**
- **Relevance** (default) - Best matches first based on search terms
- **Title (A-Z)** - Alphabetical by title
- **Date (Newest)** - Most recently published first
- **Date (Oldest)** - Historical materials first
- **Author (A-Z)** - Alphabetical by author

**Pagination:**
- 20 results per page (default)
- Page numbers at bottom
- "Previous" and "Next" buttons
- Jump to any page number

**Sharing your search:**
- **Copy Permalink button** - Creates a short URL for your exact search
- Share with colleagues or bookmark for later
- Includes all filters and sorting
- QR code available for mobile sharing

## Reading Lists

Personal organization and research management tool.

### Overview

**Reading Lists** let you:
- Organize books for research projects
- Create bibliographies for classes
- Share recommendations with others
- Track books to read
- Curate subject collections

### Creating a List

1. Navigate to **My Lists** from the catalog menu
2. Click **"+ New List"** button
3. Fill in the form:
   - **Name** - Give your list a descriptive title (required)
   - **Description** - Explain what the list is for (optional)
   - **Public/Private toggle** - Choose visibility
4. Click **"Create List"**

**List visibility:**
- **Private** - Only you can see and edit
- **Public** - Anyone can view, only you can edit

### Adding Items to a List

**From search results:**
1. Find a book you want to add
2. Click the "+ Add to List" button
3. Select which list from the dropdown
4. Item is instantly added

**From record detail page:**
1. View any book's full record
2. Click "Add to Reading List"
3. Choose destination list
4. Confirmation appears

### Managing a List

**On your list page:**

**Header shows:**
- List name and description
- Public/Private badge
- Creation date
- Number of items
- Share link button

**For each item:**
- **Full bibliographic info** - Title, author, publication
- **Personal note field** - Add your own comments or annotations
- **Remove button** - Delete from this list
- **Reorder handle** - Drag to change order

**Actions available:**
- **Edit List Details** - Change name, description, visibility
- **Delete List** - Permanently remove (with confirmation)
- **Export List** - Generate citations or bibliography
- **Share List** - Get a short URL to share with others

**Sharing a public list:**
1. Click "Share List" button
2. System generates a short URL (e.g., yourlib.com/r/abc123)
3. Copy link to clipboard
4. Share via email, social media, or embed in websites
5. QR code available for print materials

### Notes and Annotations

**For each book in your list:**
- Click "Add Note" to write personal comments
- Notes are private (even in public lists)
- Edit or delete notes anytime
- Use for:
  - Research notes
  - Page number references
  - Reading progress
  - Recommendations for others

### List Organization

**Sorting options:**
- Drag-and-drop to manually order
- Sort by title
- Sort by author
- Sort by date added
- Sort by publication date

**Best practices:**
- Use descriptive names (e.g., "American History Research - Spring 2025")
- Add descriptions so you remember the purpose
- Keep lists focused (create multiple lists for different topics)
- Make research lists public to help others

## Sharing & Permalinks

### Short URLs

Every search and record can be shared via a **short URL**.

**What can be shared:**
- Individual book records
- Search results with filters
- Reading lists (if public)

**How to create:**
1. Find the "Copy Link" or "Share" button

**Short URL format:**
- Example: `yourlib.com/r/abc123`
- 6-character code after `/r/`
- Permanent and won't change
- Works on any device

**Use cases:**

### ISBN Lookup (Copy Cataloging)

**The fastest way to catalog** - automatically import bibliographic data.

**How to use:**
1. Click **"ISBN Lookup"** from cataloging menu
2. Enter ISBN (with or without hyphens)
3. Click **"Search"**
4. System queries multiple sources
5. Review results
6. Click **"Import to Catalog"**

**Data sources (in order):**

1. **OpenLibrary API**
   - Largest online book database
   - Usually fastest
   - Comprehensive data
   - Book covers included

2. **Library of Congress**
   - Authoritative MARC records
   - High-quality cataloging
   - Subject headings standardized
   - Returns MARCXML format

**Search process (visible to user):**
- "Searching OpenLibrary..." - Step 1
- "Searching Library of Congress..." - Step 2 (if needed)
- Progress indicators shown
- Fallback if first source fails

**Imported data includes:**
- Title and subtitle
- Author(s)
- Publisher and publication date
- Physical description
- Subject headings
- Summary/description
- Book cover image (from OpenLibrary)
- ISBN (confirmed)

**After import:**
- Opens edit form with fields pre-filled
- Review and adjust as needed
- Add local information (call number, notes)
- Save to catalog

**Success rate:**
- ~90% hit rate for modern books (1990+)
- Lower for older or obscure titles
- International editions may vary
- Can fall back to manual entry

**Results table shows:**

**Best practices:**
- Limit to 50-100 ISBNs per batch
- Review imported records afterward
- Add holdings separately
- Use for new acquisitions or collection building

#### Search Behavior

**Configure how search works:**

**Default Search Type:**

- Advanced - Opens advanced search by default

**Spell Correction:**

  - `0` - Only when no results
  - `5` - When fewer than 5 results (default)
  - `10` - More lenient

**Advanced Features:**
- **Enable Advanced Search** - Show/hide advanced search link
- **Enable Boolean Operators** - Allow AND, OR, NOT in searches

### Troubleshooting Customization

**Changes don't appear:**
- Click "Save Changes" button
- Refresh your browser (Ctrl+F5 or Cmd+Shift+R)
- Clear browser cache
- Check if configuration is marked "active" in database

**Colors look wrong:**
- Verify hex codes are valid (6 characters, starts with #)
- Check color contrast for readability
- Test on different devices/browsers
- Some elements may inherit colors differently

**Custom CSS not working:**
- Ensure CSS syntax is correct
- Use browser dev tools (F12) to inspect elements
- Check for typos in class names
- More specific selectors may be needed to override defaults

**Search fields missing:**
- Check field is marked "Enabled"
- Verify "Show in Advanced Search" is checked
- Ensure field has a label
- Save configuration after changes

**Display fields not showing:**
- Verify field is enabled for the context (results/detail)
- Check if field has data in your records
- Ensure "Hide if Empty" isn't hiding fields
- Save configuration and refresh

**Preview doesn't update:**
- Make sure you've edited the field
- Try clicking elsewhere in the form
- Preview updates may have slight delay
- Refresh the admin page if needed

### Record Attachments Management

**Manage external file attachments for catalog records.**

**Access:** Cataloging ‚Üí Edit Record ‚Üí Attachments Tab

**Overview:**
The attachments system allows you to link external files to bibliographic records without storing the files in the database. This is ideal for supplementary materials, course reserves, digital collections, and subscription content with time-limited access.

#### How It Works

**Storage-Agnostic Design:**
- Files are hosted on external providers (pCloud, Amazon S3, Google Drive, Dropbox, etc.)
- ILS stores only metadata and links, not file bytes
- Staff creates share links in their storage provider
- Links are pasted into the ILS admin interface
- Patrons access files through ILS endpoints
- ILS enforces access rules and tracks analytics

#### Adding an Attachment

**From the record edit page:**

1. Click the **"Attachments"** tab
2. Click **"+ Add Attachment"**
3. Fill in the form:

**External File URL:**
- Create a share link in your storage provider (pCloud, S3, etc.)
- For temporary access, use presigned/expiring URLs
- Paste the full external URL

**Expiration (Optional):**
- Set `external_expires_at` to match your provider's link expiry
- System will check expiration before allowing downloads
- Helps manage time-limited content (course reserves, subscriptions)
- Leave empty for permanent access

**File Metadata:**
- **Title** - Descriptive name (e.g., "Teacher's Guide - Chapter 1")
- **Description** - What the file contains
- **File Type** - Select MIME type (PDF, Image, Audio, Video, etc.)
  - application/pdf
  - image/jpeg, image/png
  - audio/mpeg, video/mp4
  - text/plain
  - application/zip
  - Other
- **File Size** - Enter size (e.g., "2.5 MB")
- **Original Filename** - Original file name for reference

**Access Control:**
- **Public** - Anyone can view/download
- **Authenticated** - Requires library account login
- **Staff Only** - Librarians and staff only

**Ordering:**
- Attachments display in order of `sort_order` field

4. Click **"Save"**

#### Typical Workflows

**Course Reserves (Semester-Based):**
1. Upload file to pCloud or S3
2. Create share link with expiration (end of semester)
3. Paste URL into ILS attachment form
4. Set `external_expires_at` to match semester end
5. Set access level to "Authenticated" (students only)
6. Students access through ILS during semester
7. Link expires automatically after semester
8. Refresh URL for next semester if reusing

**Permanent Supplementary Materials:**
1. Upload to long-term storage (S3, institutional repository)
2. Create permanent share link (no expiration)
3. Paste URL into ILS
4. Leave `external_expires_at` empty
5. Set appropriate access level (Public, Authenticated, Staff)
6. Material remains available indefinitely
7. Update URL if storage location changes

**Subscription Content:**
1. Link to vendor-provided content
2. Set expiration to match subscription end date
3. Set access level to "Authenticated" or "Staff Only"
4. Monitor usage via download counts
5. Update when subscription renews

**Digital Special Collections:**
1. Store high-res images/documents in repository
2. Create public share links
3. Add descriptive metadata
4. Set access level to "Public" for open access
5. Track views and downloads for impact reporting

#### Access Control & Security

**How access levels work:**

- **Public:** No authentication required
  - Anyone with the link can access
  - Good for open educational resources
  - Community materials

- **Authenticated:** Library account required
  - User must log in to ILS
  - Verified against patron database
  - Good for licensed content, course reserves

- **Staff Only:** Librarian accounts only
  - Restricted to staff patron types
  - Staff, faculty, librarian, or admin roles
  - Good for internal documents, procedures

**Download flow:**
1. Patron clicks "Download" on public record page
2. Request goes to `/api/attachments/{id}/download`
3. ILS checks:
   - User's authentication status
   - Access level permissions
   - Expiration date (if set)
4. If authorized and not expired:
   - Download count incremented
   - User redirected (302) to external URL
   - File downloads from provider
5. If unauthorized:
   - 401 error (authentication required)
6. If expired:
   - 410 error (gone/expired)

**Security notes:**
- External URLs should be from trusted providers
- Expiring URLs recommended for sensitive content
- Staff-only for confidential materials
- Never store passwords or API keys in ILS
- Provider handles actual file security

#### Expiration Management

**Why use expiration:**
- Course reserves (semester-based access)
- Temporary exhibits or events
- Subscription content (annual renewals)
- Comply with licensing restrictions
- Storage provider link expiration

**Setting expiration:**
1. Create expiring share link in provider (e.g., pCloud presigned URL)
2. Note the expiration timestamp
3. Enter matching timestamp in `external_expires_at` field
4. ILS enforces expiration on download requests

**Expiration behavior:**
- System checks expiration before each download
- Expired attachments show ‚ùå **Expired** badge
- Download button disabled on public page
- Returns 410 error if download attempted
- Staff can still see expired attachments (for renewal)

**Renewing expired attachments:**
1. Create new share link in provider
2. Edit attachment in ILS
3. Update `external_url` with new link
4. Update `external_expires_at` if needed
5. Save changes
6. Attachment becomes available again

#### Troubleshooting

**"External URL not accessible"**
- Check URL in browser
- Verify share permissions in provider
- Ensure URL hasn't expired
- Regenerate share link if needed

**"Download button disabled"**
- Check expiration date (may be expired)
- Verify access level matches user status
- Check if user is logged in (if authenticated required)

**"File won't download"**
- Provider may have rate limits
- Storage quota may be exceeded
- File may have been deleted from provider
- Update URL with new location

**"Analytics not updating"**
- Views increment on list load
- Downloads increment on download request
- May have caching delay
- Check database directly if needed
